# Contributing

Thank you for your interest in contributing to this project.

This document provides guidelines and instructions to help you get started and ensure a consistent contribution process.

## Getting Started

1. Fork the repository
2. Clone your fork locally
3. Create a new branch for your changes

```bash
git checkout -b feature/your-feature-name
````

## Development Guidelines

* Keep changes focused and minimal
* Follow the existing code style and structure shown in the eq-wiki.github.io
* Write clear, maintainable code
* Add comments where necessary
* Ensure your changes do not break existing functionality

## Commit Messages

Use clear and descriptive commit messages.

Recommended format:

```
type(scope.subscope): short description
```

Examples:

* `feat(ui.controlcenter): add glass blur effect`
* `fix(parser): handle empty input correctly`
* `docs(readme): update installation instructions`

## Reporting Issues

If you find a bug, please use the bug report template provided in the repository.

Include:

* A clear description
* Steps to reproduce
* Expected and actual behavior
* Relevant logs or screenshots

## Suggesting Features

If you have an idea for a new feature, please use the feature request template.

Provide:

* A clear problem statement
* A proposed solution
* Relevant use cases

## Pull Requests

Before submitting a pull request:

* Ensure your branch is up to date with the main branch
* Verify that your changes work as intended
* Check for unnecessary files or changes

When submitting:

* Provide a clear description of your changes
* Reference related issues if applicable
* Keep pull requests focused on a single topic

## Code Review

All contributions are reviewed before merging.

Feedback may include:

* Requested changes
* Suggestions for improvement
* Questions for clarification

Please respond to feedback in a timely and constructive manner.

## Code of Conduct

By participating in this project, you agree to follow the Code of Conduct.

## Questions

If you have any questions, feel free to open an issue or start a discussion.
